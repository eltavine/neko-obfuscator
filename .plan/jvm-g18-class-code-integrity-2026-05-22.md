# JVM G18 Class-Code Integrity - 2026-05-22

## Scope

Implement generic JVM full-obfuscation anti-tamper enforcement for protected
class method bodies during class key-table initialization. The target bypass is
method-body instrumentation of protected methods while leaving the G18 host,
class initializer, and key table fields untouched.

## Status Legend

- `[ ]` not implemented or not freshly verified
- `[-]` active subtask
- `[x]` implemented, freshly validated, and committed

## Constraints

- Do not special-case any sample jar, owner, method, descriptor, password, seed,
  flag, or known generated name.
- Do not add new classes, JNI, JVMTI, fallback, skip-on-error, or original
  bytecode fallback.
- Keep checks on the existing JVM CFF/G18/key-table surface.
- A patched method body must fail closed before protected CFF, constant, string,
  or invokedynamic material remains usable.
- Use repository `./gradlew` only after explicit permission.
