# ZKM Global g18 Correction - 2026-05-15

## Scope

Correct the prior per-class g18-shaped implementation so class key initialization is distributed through one generated global g18 host, not one independent g18 cell per transformed class.

## Evidence

- Generated bytecode currently shows each class allocates its own carrier-local g18 cell: `objectArray3[78] = new AtomicLong(...)` and immediately projects/mutates that same per-class cell.
- That is not equivalent to ZKM's global `g18` registry/table model where classes request subkey material from a shared distributor.
- The previous plan's "global-state host" completion text was therefore over-claimed: the behavior was class-local, not global.

## Required invariants

- One generated global g18 host per obfuscation output, stored on a generated application-owned host class.
- Every protected class must obtain its class root by calling/reading that global host; no protected class may allocate its own g18 `AtomicLong` state cell in its local class carrier.
- The global host must own the mutable registry/table state and class-indexed node table used to distribute class subkeys.
- Class-root projection must happen before mutation; host/global state and the class node must be mutated after emission.
- Existing behavior for the four full-obf jars must be preserved.
- No fallback, skip, original-bytecode path, split hidden-key ABI, or per-class g18 fallback is allowed.

## Subtasks

### [ ] Subtask 1: Map current carrier and init paths

- Scope: inspect `CffClassSetup`, `CffMaterialTables`, and generated TEST bytecode to identify every per-class g18 allocation/emission site.
- Required evidence: exact source sites for carrier slot allocation, g18 state creation, root projection, and class word decode.
- Validation target: source audit plus generated bytecode observation.
- Completion criteria: implementation targets are known and no unexamined per-class g18 path remains.

### [ ] Subtask 2: Add one global g18 host contract

- Scope: extend CFF shared metadata with a generated global host field/helper and stable per-class global g18 index assignment.
- Required evidence: one host owner/field/helper is reused by all class tables across packages/classes.
- Validation target: compile and source audit.
- Completion criteria: protected classes can refer to a single shared g18 distributor without local g18 state allocation.

### [ ] Subtask 3: Rebind class-root decode to global host

- Scope: replace local `G18_CLASS_STATE_SLOT` initialization/projection with a call into the global host distributor and decode class words from the returned root.
- Required evidence: generated class `<clinit>` contains no local g18 `AtomicLong` slot allocation; it calls/reads the shared host and decodes class words from that root.
- Validation target: full-obf TEST `javap`/decompile inspection.
- Completion criteria: all protected class subkey tables are issued by the shared global g18 host.

### [ ] Subtask 4: Validate behavior and structural invariants

- Required evidence: `JvmFullObfuscationPerfTest` passes; generated-output scan rejects local g18 slot patterns and split/fallback markers.
