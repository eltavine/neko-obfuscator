# CFF/InvokeDynamic/String dynamic-flowkey continuation plan

## Non-negotiable constraints

- All accepted changes must preserve strong runtime dependency on live flowkey material.
- No static decrypt, static key recovery, descriptor-only recomputation, static registry decrypt, original-bytecode fallback, soft fallback, skip-on-error path, JNI fallback, or reduced obfuscation coverage is allowed.
- Hidden method keys and packed `Object[]` carriers remain mandatory obfuscation surfaces.
- Control-flow flattening block construction, block boundaries, block count, and block selection must not be weakened to make a jar pass.
- Validation is limited to JVM-obfuscation-related targeted tests only; no full test suite.

## Current continuation state

- Accepted baseline keeps `TOKEN_MATERIAL_INIT_CHUNK_SIZE = 128` and `SMALL_TOKEN_DISPATCH_CASES = 4`.
- Accepted baseline keeps per-class invoke-dynamic bootstrap trampolines; the rejected shared-BSM registry route is not present.
- Accepted baseline keeps caller-local string carriers for non-`<clinit>` ordinary string decode and loop-externalized concat decode.
